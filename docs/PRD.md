# PRD — 반도체 시료 생산주문관리 시스템

## 1. 배경 및 목적

가상의 반도체 회사 **S-Semi**는 다양한 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체, 대학 연구실 등에 납품한다. 시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.

주문량 급증으로 엑셀·메모장 기반 관리의 한계가 드러나면서 아래 문제가 반복되었다:
- 주문 처리 여부 불명확
- 공정 완료 시점 파악 불가
- 재고가 충분함에도 불필요한 추가 공정 가동

이를 해결하기 위해 **콘솔 기반 반도체 시료 생산주문관리 시스템**을 개발한다.

---

## 2. 이해관계자 및 역할

| 역할 | 설명 | 주요 행위 |
|------|------|-----------|
| 고객 | 시료 요청자 | 필요한 시료를 이메일로 요청 |
| 주문 담당자 | 주문서 관리 | 요청에 맞게 주문서 작성 및 생산 담당자에게 전달 |
| 생산 담당자 | 시료 생산·승인 | 개발 시료 등록, 주문 수신 후 승인 또는 거절 |

---

## 3. 시스템 개요

- **인터페이스:** 콘솔(CLI) 기반, 담당자가 직접 명령 입력
- **생산 라인:** 단일 라인, 시료를 하나씩 순차 생산(FIFO), 주문이 들어온 시료에 대해서만 생산

---

## 4. 주문 상태 흐름

```
[주문 등록]
    RESERVED
        │
        ├─ 거절 ──────────────────→ REJECTED  (모니터링 제외)
        │
        └─ 승인
              │
              ├─ 재고 충분 ────────→ CONFIRMED (출고 대기)
              │                          │
              │                          └─ 출고 ──→ RELEASE
              │
              └─ 재고 부족 ────────→ PRODUCING (생산 중)
                                         │
                                   생산 완료
                                         │
                                     CONFIRMED
                                         │
                                       출고
                                         │
                                      RELEASE
```

| 상태 | 의미 |
|------|------|
| `RESERVED` | 주문 접수 |
| `REJECTED` | 주문 거절 (정상 흐름 외, 모니터링 제외) |
| `PRODUCING` | 주문 승인 완료 + 재고 부족으로 생산 중 |
| `CONFIRMED` | 주문 승인 완료 + 출고 대기 중 |
| `RELEASE` | 출고 완료 |

## 5. 기능 명세

### 5-1. 메인 메뉴

시작 시 전체 시료 요약 정보(등록 시료 수, 총 재고, 전체 주문 수, 생산라인 대기 수)를 표시하고 메뉴를 출력한다.

| 번호 | 메뉴 | 설명 |
|------|------|------|
| 1 | 시료 관리 | 시료 등록, 목록 조회, 이름 검색 |
| 2 | 시료 주문 | 고객 주문 접수 |
| 3 | 주문 승인/거절 | 접수된 주문에 대한 승인·거절 처리 |
| 4 | 모니터링 | 상태별 주문 수 및 시료별 재고 현황 |
| 5 | 생산라인 조회 | 현재 생산 중인 시료 및 대기 큐 확인 |
| 6 | 출고 처리 | CONFIRMED 주문에 대해 출고 실행 |
| 0 | 종료 | 시스템 종료 |

---

### 5-2. 시료 관리

시료(Sample)는 시스템의 기본 단위이며, 등록된 시료만 주문 가능하다.

#### 시료 등록

| 속성 | 타입 | 설명 |
|------|------|------|
| 시료 ID | String | 고유 식별자 (예: `S-001`) |
| 이름 | String | 시료명 (예: `실리콘 웨이퍼-8인치`) |
| 평균 생산시간 | double | 시료 1개당 평균 생산 소요 시간 (단위: min/ea) |
| 수율 | double | 정상 시료 / 총 생산 시료 (예: 0.92) |

- 시료 ID는 중복 등록 불가

#### 시료 조회

- 등록된 모든 시료 목록 출력
- 각 시료의 현재 재고 수량 함께 표시

#### 시료 검색

- 이름 등 속성으로 특정 시료 검색

---

### 5-3. 시료 주문 (접수)

고객이 원하는 시료와 수량을 주문한다. 주문 접수 시 상태는 `RESERVED`.

**입력 값:**
- 시료 ID
- 고객명
- 주문 수량

**주문번호 형식:** `ORD-YYYYMMDD-NNNN` (예: `ORD-20260416-0043`)

입력 확인 화면을 보여주고 예약 접수 여부를 확인받는다.

---

### 5-4. 주문 승인/거절

`RESERVED` 상태의 주문 목록을 표시하고, 담당자가 개별 주문에 대해 승인 또는 거절을 처리한다.

#### 주문 승인

재고 상황에 따라 자동으로 2가지 경로로 처리된다:

| 조건 | 처리 |
|------|------|
| 재고 ≥ 주문 수량 | 즉시 `CONFIRMED`로 전환 |
| 재고 < 주문 수량 | 생산 라인에 자동 등록 + `PRODUCING`으로 전환 |

승인 시 부족분, 실생산량, 예상 생산 시간을 함께 표시한다.

#### 주문 거절

즉시 `REJECTED` 상태로 전환.

---

### 5-5. 모니터링

담당자가 시스템 전체 상태를 한눈에 파악할 수 있는 화면.

#### 주문량 확인

상태별 주문 건수 표시 (`REJECTED` 제외):

| 상태 | 건수 |
|------|------|
| RESERVED | N건 |
| CONFIRMED | N건 |
| PRODUCING | N건 (← 생산라인 대기) |
| RELEASE | N건 |

#### 재고량 확인

각 시료별 현재 재고 수량 및 재고 상태 표기:

| 상태 | 조건 |
|------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 재고 수량 = 0 |

---

### 5-6. 생산 라인

단일 생산 라인에서 FIFO 방식으로 주문을 처리한다.

#### 생산량 계산

```
실 생산량 = ceil(부족분 / (수율 × 0.9))
총 생산 시간 = 평균 생산시간 × 실 생산량
```

> 수율에 0.9를 곱하는 이유: 공정 오차 버퍼를 반영하여 실제 요구량을 충족할 수 있도록 여유분 생산

#### 생산 완료 처리

생산 완료 시 주문 상태: `PRODUCING` → `CONFIRMED`

#### 생산 현황 표기

현재 생산 중인 시료에 대해 다음 정보를 표기:
- 주문번호, 시료명
- 주문량, 기존 재고, 부족분, 실생산량
- 진행률(%), 완료 예정 시각

#### 대기 주문 확인

생산 큐에 대기 중인 주문 목록 출력 (FIFO 순):
- 순서, 주문번호, 시료명, 주문량, 부족분, 실생산량, 예상 완료 시각

---

### 5-7. 출고 처리

`CONFIRMED` 상태의 주문 목록을 표시하고 특정 주문에 대해 출고를 실행한다.

- 출고 실행 시 주문 상태: `CONFIRMED` → `RELEASE`
- 출고 결과: 주문번호, 출고 수량, 처리 일시, 상태 변경 표시

## 6. 데이터 모델

### Sample (시료)

| 필드 | 타입 | 설명 |
|------|------|------|
| `sampleId` | String | 고유 ID (예: `S-001`) |
| `name` | String | 시료명 |
| `avgProductionTime` | double | 평균 생산시간 (min/ea) |
| `yield` | double | 수율 (0 < yield ≤ 1) |
| `stock` | int | 현재 재고 수량 |

### Order (주문)

| 필드 | 타입 | 설명 |
|------|------|------|
| `orderId` | String | 주문번호 (`ORD-YYYYMMDD-NNNN`) |
| `sampleId` | String | 주문 시료 ID |
| `customerName` | String | 고객명 |
| `quantity` | int | 주문 수량 |
| `status` | OrderStatus | 주문 상태 |
| `createdAt` | LocalDateTime | 주문 접수 일시 |

### ProductionJob (생산 작업)

| 필드 | 타입 | 설명 |
|------|------|------|
| `orderId` | String | 연결된 주문번호 |
| `sampleId` | String | 생산 시료 ID |
| `shortage` | int | 부족분 수량 |
| `actualProductionQty` | int | 실 생산량 (`ceil(shortage / (yield * 0.9))`) |
| `totalProductionTime` | double | 총 생산 시간 (min) |
| `enqueuedAt` | LocalDateTime | 큐 등록 일시 |

---

## 8. 제약 사항

- 시스템에 등록된 시료만 주문 가능
- `REJECTED` 주문은 모니터링 대상에서 제외
- 생산 라인은 단일 라인으로, 동시에 하나의 작업만 처리
- 주문 ID는 날짜 + 순번으로 자동 생성
