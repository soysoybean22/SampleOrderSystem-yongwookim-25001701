# Phase 8 설계 — 통합 및 마무리

## 1. 목표

Phase 2~7을 통해 모든 메뉴 기능은 이미 구현되었다.  
Phase 8은 **전체 흐름 검증 + 마무리 정리**가 목표다.

구체적으로 다음 세 가지를 수행한다:

1. **MainView 요약 정보 보완** — 메인 화면에서 RESERVED(승인 대기) 건수를 추가해 담당자가 처리해야 할 주문이 있는지 한눈에 파악 가능하도록 개선
2. **잘못된 입력 처리 최종 확인** — 모든 메뉴에서 숫자/문자 오입력 시 재입력 요청 동작 확인
3. **전체 흐름 통합 시나리오 테스트** — 처음부터 끝까지 하나의 시나리오로 검증

---

| 확인 | 통합 시나리오 | 아래 시나리오 A, B 수동 실행 |

## 4. 통합 시나리오

### 시나리오 A — 재고 충분 경로 (End-to-End)

| 순서 | 메뉴 | 행동 | 기대 결과 |
|------|------|------|-----------|
| 1 | 초기화 | `dummy` 실행 | 시료 5종, 주문 10건 생성 |
| 2 | `[1]` | 새 시료 등록 (S-006, InP 기판-2인치, 1.2, 0.85, 50) | 등록 완료 |
| 3 | `[2]` | S-001 / 고객명 / 100 주문 접수 | RESERVED 주문 생성 |
| 4 | `[3]` | 위 주문 승인 | 재고 충분 → CONFIRMED, 재고 차감 |
| 5 | `[4]→[1]` | 주문량 확인 | CONFIRMED 증가 확인 |
| 6 | `[6]` | 위 주문 출고 | CONFIRMED → RELEASE |
| 7 | `[4]→[1]` | 주문량 확인 | RELEASE 증가, CONFIRMED 감소 |
| 8 | `[0]` | 종료 후 재실행 | 모든 데이터 유지 (영속성) |

### 시나리오 B — 재고 부족 경로 (End-to-End)

| 순서 | 메뉴 | 행동 | 기대 결과 |
|------|------|------|-----------|
| 1 | `[2]` | S-003 / 고객명 / 200 주문 접수 (재고 30) | RESERVED 생성 |
| 2 | `[3]` | 위 주문 승인 | 재고 부족 → PRODUCING, 생산 큐 등록 |
| 3 | `[5]` | 생산라인 조회 | 현재 처리 중 작업 표시 확인 |
| 4 | `[5]` | 생산 완료 처리 | PRODUCING → CONFIRMED, 재고 증가 |
| 5 | `[4]→[2]` | 재고량 확인 | S-003 재고 증가 확인 |
| 6 | `[6]` | 출고 처리 | CONFIRMED → RELEASE |
| 7 | `[4]→[1]` | 주문량 확인 | 전체 흐름 건수 최종 확인 |

## 5. 잘못된 입력 처리 확인 목록

| 위치 | 입력 | 기대 동작 |
|------|------|-----------|
| 메인 메뉴 | 숫자 범위 외 (`9`, `abc`) | "잘못된 입력" 후 메뉴 재출력 |
| 시료 등록 | 평균 생산시간에 문자 입력 | "숫자를 입력해주세요" 후 재입력 |
| 시료 등록 | 수율에 `0` 또는 `1.5` 입력 | `IllegalArgumentException` → "오류" 메시지 |
| 주문 접수 | 없는 시료 ID | "등록되지 않은 시료 ID" 오류 |
| 주문 접수 | 수량 `0` | "수량은 1 이상" 오류 |
| 승인/거절 | 목록 외 번호 | "목록에 없는 번호" 오류 |
| 출고 처리 | 목록 외 번호 | "목록에 없는 번호" 오류 |
